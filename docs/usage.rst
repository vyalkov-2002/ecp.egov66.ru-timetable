Использование
=============

В этом разделе говорится о том, как использовать библиотеку в ваших проектах, а
также разбираются некоторые продвинутые возможности.


Клиент
------

Единичные запросы удобно делать с помощью клиента :class:`Client
<egov66_timetable.client.Client>` и :class:`TeacherClient
<egov66_timetable.client.TeacherClient>`.

Если настройки хранятся в файле :file:`settings.json`, то получить расписание
для группы 101 на следующую неделю можно следующим образом:

.. code-block:: python

   from egov66_timetable.client import Client
   from egov66_timetable.utils import (
       read_settings,
       write_settings,
   )

   settings = read_settings()
   client = Client(settings)
   timetable = client.make_timetable("101", offset=offset)

   ...

   # Нужно обязательно обновить cookie
   write_settings(settings)


Расписание преподавателя
------------------------

Расписание преподавателя отличается от расписания студента. Вместо номера
аудитории в нем будет номер группы, а еще в одно время у одного преподавателя
могут быть занятия у нескольких групп, поэтому тип данных будет
``Timetable[list[Lesson]]`` вместо ``Timetable[Lesson]``.


Коллбэки
--------

Когда вы уже освоитесь с единичными запросами, можно переходить к запросу
всего доступного расписания. Для этого вам придется составить список всех групп
и преподавателей, которые есть в личном кабинете. Задачка не из легких, но вы
справитесь.

Функции :func:`get_timetable <egov66_timetable.get_timetable>` и
:func:`get_teacher_timetable <egov66_timetable.get_teacher_timetable>` позволяют
получить несколько расписаний и вызвать коллбэк-функции с каждым из них.

Коллбэк-функция для расписания студента принимает 3 аргумента: расписание, номер
группы и неделю.

В комплекте с библиотекой идет несколько коллбэков.

Генерация HTML
``````````````

Используйте функции :func:`html_callback
<egov66_timetable.callbacks.html.html_callback>` и :func:`html_teacher_callback
<egov66_timetable.callbacks.html.html_teacher_callback>`, чтобы сгенерировать
статические HTML-файлы с расписанием.

Вы можете использовать свой шаблон или оставить шаблон по умолчанию. Путь, по
которому браузер запрашивает CSS-стили, вы можете задать в настройках.

Кроме коллбэков есть полезная функция :func:`collapse_timetable
<egov66_timetable.callbacks.html.collapse_timetable>` и
:func:`collapse_teacher_timetable
<egov66_timetable.callbacks.html.collapse_teacher_timetable>`, которая
пригодится для любого табличного отображения данных, где ячейки можно
объединить.

База данных SQLite
``````````````````

Создайте базу данных с помощью функции :func:`create_db
<egov66_timetable.callbacks.sqlite.create_db>`.

Используйте функции :func:`sqlite_callback
<egov66_timetable.callbacks.sqlite.sqlite_callback>` и
:func:`sqlite_teacher_callback
<egov66_timetable.callbacks.sqlite.sqlite_teacher_callback>`, чтобы записать
расписание в базу данных.

Загрузите расписание из базы данных с помощью функции :func:`load_timetable
<egov66_timetable.callbacks.sqlite.load_timetable>`.

Другие коллбэки
```````````````

В проекте :doc:`ИАТ-Расписание <showcase>` коллбэк-функция используется для
отправки уведомлений о новом расписании в мессенджерах.

Номер аудитории
---------------

Библиотека не возвращает номер аудитории, если в нем больше трех символов.
Возможно, стоит сделать этот параметр настраиваемым. Жду ваши патчи :)


Алиасы
------

Название учебной дисциплины в личном кабинете может быть длинным и непонятным. У
нескольких предметов может быть одно и то же название. Чтобы решить эту
проблему, библиотека позволяет указать свои алиасы (переименования).

Список переименований передается в словаре :class:`Settings
<egov66_timetable.types.settings.Settings>`:

.. code-block:: python

   {
     ...

     "aliases": [
       {...},
       {...}
      ]
   }

Каждый алиас представляет собой словарь типа :class:`Alias
<egov66_timetable.types.settings.Alias>`. Обязательными являются поля
**discipline** и **rename**, которые означают исходное название и новое название
соответственно.

Итоговое название будет определено в порядке предпочтения:

1. Поле "Комментарий" в исходных данных
2. Алиас, выбранный по совпадению названия учебной дисциплины и ФИО
   преподавателя
3. Алиас, выбранный по совпадению названия учебной дисциплины и номера аудитории
4. Алиас, выбранный только по названию (простое переименование)
5. Исходное название

В одном алиасе можно указать и ФИО преподавателя, и номер аудитории, это не
возбраняется.
